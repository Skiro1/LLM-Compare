📡 API Documentation

Welcome to the Multi-LLM API documentation. The API provides a single interface for working with different language models through a single query format compatible with the OpenAI API.

💡 Unified format

All providers use a format compatible with the OpenAI API. This makes it easy to switch between models without changing the code.

Key features

  • Access 20+ language models through a single API
  • Support for chat format with system prompts
  • Streaming responses
  • JSON mode for structured responses
  • Compatible with OpenAI SDK

Your first request

Here is a minimal example of sending a message to a model:

cURL
curl https://api.openai.com/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "gpt-4",
    "messages": [
      {"role": "system", "content": "You are a helpful assistant."},
      {"role": "user", "content": "Hello! Tell me about yourself."}
    ]
  }'
import requests

response = requests.post(
    "https://api.openai.com/v1/chat/completions",
    headers={
        "Content-Type": "application/json",
        "Authorization": "Bearer YOUR_API_KEY"
    },
    json={
        "model": "gpt-4",
        "messages": [
            {"role": "system", "content": "You are a helpful assistant."},
            {"role": "user", "content": "Hello! Tell me about yourself."}
        ]
    }
)

data = response.json()
print(data["choices"][0]["message"]["content"])
const response = await fetch("https://api.openai.com/v1/chat/completions", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "Authorization": "Bearer YOUR_API_KEY"
  },
  body: JSON.stringify({
    model: "gpt-4",
    messages: [
      {role: "system", content: "You are a helpful assistant."},
      {role: "user", content: "Hello! Tell me about yourself."}
    ]
  })
});

const data = await response.json();
console.log(data.choices[0].message.content);

🔑 Authentication

All API requests require authentication via an API key. The key is passed in the Authorization header.

HEADER Authorization: Bearer YOUR_API_KEY

Replace YOUR_API_KEY with your actual API key from the provider.

Getting API keys

Each provider has its own way to obtain an API key:

Provider Where to get the key Key format
OpenAI platform.openai.com sk-...
Anthropic 🔒 proxy console.anthropic.com sk-ant-...
Google aistudio.google.com AIza...
DeepSeek platform.deepseek.com sk-...
Mistral console.mistral.ai ...
Groq console.groq.com gsk_...
Hugging Face huggingface.co hf_...
OpenRouter openrouter.ai sk-or-...
⚠️ Security

Never store API keys in client-side code. Use environment variables or server-side proxies for production applications.

Import and export of keys

In the settings (the "Providers" tab), buttons for bulk key management are available:

  • Import keys (.txt) — upload a file with keys, all providers at once
  • Export keys — download all keys to a api-keys.txt file
  • Delete all keys — clear all keys with one click (with confirmation)

💬 Chat Completions

Creates a model response for a given conversation.

POST /v1/chat/completions

Request parameters

model string Required
Model ID to use. For example: gpt-4, claude-3-opus, gemini-pro
messages array Required
List of messages representing the conversation. Each object contains role and content.
temperature number Optional
Sampling temperature from 0 to 2. Higher values make responses more random. Default: 1
max_tokens integer Optional
Maximum number of tokens in the response. Default: model-dependent
stream boolean Optional
Enable response streaming. Default: false
response_format object Optional
Response format. Use {"type": "json_object"} for JSON mode

Message structure

Each message in the messages array contains:

Role Description
system System prompt that defines the model's behavior
user Message from the user
assistant Previous model response (for context)

Response example

{ "id": "chatcmpl-abc123", "object": "chat.completion", "created": 1699876543, "model": "gpt-4", "choices": [ { "index": 0, "message": { "role": "assistant", "content": "Hello! I am a language model developed by OpenAI..." }, "finish_reason": "stop" } ], "usage": { "prompt_tokens": 25, "completion_tokens": 150, "total_tokens": 175 } }

📋 List Models

Returns a list of all available models.

GET /v1/models

Does not require parameters. Returns a list of all models available for your API key.

Request example

curl https://api.openai.com/v1/models \
  -H "Authorization: Bearer YOUR_API_KEY"
import requests

response = requests.get(
    "https://api.openai.com/v1/models",
    headers={"Authorization": "Bearer YOUR_API_KEY"}
)

models = response.json()["data"]
for model in models:
    print(model["id"])

Response example

{ "object": "list", "data": [ { "id": "gpt-4", "object": "model", "created": 1687882411, "owned_by": "openai" }, { "id": "gpt-3.5-turbo", "object": "model", "created": 1677610602, "owned_by": "openai" } ] }

🔍 Retrieve Model

Returns information about a specific model.

GET /v1/models/{model}

Path parameters

model string Required
Model ID. For example: gpt-4

Request example

curl https://api.openai.com/v1/models/gpt-4 \
  -H "Authorization: Bearer YOUR_API_KEY"

⚡ Streaming

Streaming allows you to receive the model's response as it is generated, improving perceived response speed.

💡 When to use streaming

Use streaming for long responses or when it's important to show the user the generation progress in real time.

Enabling streaming

Add the "stream": true parameter to the request:

curl https://api.openai.com/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "gpt-4",
    "messages": [{"role": "user", "content": "Tell a long story"}],
    "stream": true
  }'
import requests

response = requests.post(
    "https://api.openai.com/v1/chat/completions",
    headers={
        "Content-Type": "application/json",
        "Authorization": "Bearer YOUR_API_KEY"
    },
    json={
        "model": "gpt-4",
        "messages": [{"role": "user", "content": "Tell a long story"}],
        "stream": True
    },
    stream=True
)

for line in response.iter_lines():
    if line:
        line = line.decode("utf-8")
        if line.startswith("data: ") and line != "data: [DONE]":
            import json
            data = json.loads(line[6:])
            content = data["choices"][0]["delta"].get("content", "")
            print(content, end="", flush=True)
const response = await fetch("https://api.openai.com/v1/chat/completions", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "Authorization": "Bearer YOUR_API_KEY"
  },
  body: JSON.stringify({
    model: "gpt-4",
    messages: [{role: "user", content: "Tell a long story"}],
    stream: true
  })
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  
  const chunk = decoder.decode(value);
  const lines = chunk.split("\n").filter(l => l.startsWith("data: "));
  
  for (const line of lines) {
    if (line === "data: [DONE]") break;
    const data = JSON.parse(line.slice(6));
    const content = data.choices[0].delta.content || "";
    process.stdout.write(content);
  }
}

SSE response format

Each chunk comes in Server-Sent Events format:

data: {"id":"chatcmpl-123","object":"chat.completion.chunk","choices":[{"delta":{"content":"Hello"},"index":0}]}

data: {"id":"chatcmpl-123","object":"chat.completion.chunk","choices":[{"delta":{"content":" world!"},"index":0}]}

data: [DONE]

❌ Errors

The API uses standard HTTP error codes. When an error occurs, the response body contains a detailed description.

Code Description Solution
400 Bad Request Check the request format and parameters
401 Unauthorized Check the API key
403 Forbidden No access to the requested resource
404 Not Found Check the URL and model ID
429 Rate Limited Request limit exceeded. Wait and retry
500 Server Error Server-side error. Try again later
503 Service Unavailable Service temporarily unavailable

Error format

{ "error": { "message": "Incorrect API key provided: sk-xxx...xxx", "type": "invalid_request_error", "param": null, "code": "invalid_api_key" } }

Retry logic

✅ Best Practice

When receiving a 429 (Rate Limit) or 5xx (Server Error) error, use exponential backoff: wait 1 second, then 2, 4, 8... Maximum 5 attempts.

⏱️ Rate Limits

Request limits depend on the provider and your pricing plan. Limit information is passed in the response headers.

Response headers

Header Description
x-ratelimit-limit-requests Maximum number of requests per period
x-ratelimit-remaining-requests Remaining number of requests
x-ratelimit-reset-requests Limit reset time (RFC 7231)
x-ratelimit-limit-tokens Maximum number of tokens per period
x-ratelimit-remaining-tokens Remaining number of tokens

🔀 OpenRouter

OpenRouter is an API aggregator that provides access to 300+ models from different providers through a unified interface compatible with the OpenAI API. One key, all models.

Request example к OpenRouter

curl https://openrouter.ai/api/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer sk-or-YOUR_TOKEN" \
  -d '{
    "model": "anthropic/claude-3.5-sonnet",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'

OpenRouter Benefits

  • One API key to access models from OpenAI, Anthropic, Google, Meta, Mistral and others
  • Automatic fallback between providers
  • Unified billing and expense tracking
  • Free models available without card binding
